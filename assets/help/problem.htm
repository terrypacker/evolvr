Built-in Problems

Peak Finder

The genome encodes X and Y coordinates in a 2D search space filled with hidden peaks of varying heights. Organisms must find the tallest peak. The visualization shows the landscape as a heat map — brighter areas are higher — with organisms shown as coloured dots. The best organism gets a highlighted ring.

Explorer and Mutant types tend to perform well here because the space is multi-modal — there are many local peaks to escape.
Function Approximation

The genome encodes polynomial coefficients. Organisms try to match a target sine-like curve over the interval [0, 1]. Fitness is based on how closely the polynomial matches the target. The visualization shows the target curve in amber and each organism's curve overlaid.

Optimizer and Climber types do well here as the problem rewards precision over exploration.
Binary Knapsack

Each genome bit decides whether to include a corresponding item in a virtual knapsack. The goal is to maximise total value without exceeding the weight capacity. Cyan-outlined bars show which items the best organism selected. The red dashed line marks the capacity limit.

A mixed population of Optimizers and Climbers tends to find good packing strategies quickly.
Traveling Salesman

The genome encodes a visit order for a set of cities. Organisms evolve the shortest possible tour visiting all cities and returning to the start. The best tour is shown in amber; other population tours are shown faintly in their type colour.

This problem is NP-hard, so a diverse mix of all four types works best — Mutants escape local optima while Climbers refine short routes.
🔧 To add your own problem, open problems.js and push a new object into the Problems array. It needs an id, label, evaluate(genome, expressed, organism) function returning 0–1, and a visualize(canvas, population, goal) function. The UI picks it up automatically.

How Each Problem Evaluates Fitness

Peak Finder — Gaussian Mixture

The landscape is a sum of 7 Gaussian "hills", each centred at a fixed point (xₚ, yₚ) in [0,1]². The height function is:

h(x, y) = min(1, Σₚ hₚ · exp( −(x−xₚ)² + (y−yₚ)² / 2σₚ² ))

Fitness is simply h(expressed[0], expressed[1]) — the landscape height at the organism's expressed (x, y) coordinates. Because fitness depends only on the first two expressed values, genes that transform positions 2 and beyond have no effect at all. The optimal gene strategy is one that helps the organism zero-in on a high-value (x, y) point without disrupting those two coordinates.

Good genes for this problem: randomWalk and gaussianNoise create small spatial movements around the current position. boundaryPush prevents corners of the search space from being over-represented. Genes that preserve the magnitude of positions (rather than remapping them) work best.

Problematic genes: mirrorFold forces expressed[1] = 1 − expressed[0], which constrains organisms to the anti-diagonal of the search space, making most peaks unreachable. rankSort and normalize destroy positional information by replacing values with their relative rank.

Function Approximation — Polynomial MSE

The expressed genome is interpreted as the coefficients of a polynomial p(x) = Σᵢ aᵢ · xⁱ, evaluated over 20 equally-spaced sample points x ∈ {0, 1/19, 2/19, …, 1}. The target function is:

f(x) = 0.5 · sin(2πx) + 0.5

The variance (VAR) is computed as VAR = (1/20) · Σᵢ f(xᵢ))² and the mean squared error (MSE) is MSE = = (1/20) · Σᵢ (p(xᵢ) − f(xᵢ))². Fitness is then 1 - (MSE / variance) bounded to [0,1], meaning perfect fit scores 1.0 and an average squared deviation of 0.05 scores 0. The coefficient aᵢ at genome position i is multiplied by xⁱ — so later genome positions (high-degree terms) control fine wiggles in the curve, while early positions (low-degree terms, especially a₀) set the baseline level.

Good genes for this problem: Genes that preserve the continuous, real-valued nature of coefficients work well. gradientNudge and elitePull gently converge coefficient values, reducing overshoot. sinTransform compresses coefficients toward 0.5, which suppresses higher-degree oscillation — useful early in evolution. The polynomial is clipped to [0,1], so extreme coefficient values are automatically penalised.

Problematic genes: rankSort replaces coefficient values with their ordinal rank, which destroys the numeric meaning of each coefficient entirely. bitFlip introduces large random coefficient changes that tend to produce wildly oscillating polynomials far from the smooth target.

Binary Knapsack — Threshold Decoding

There are 12 items, each with a fixed weight wᵢ ∈ [0.05, 0.30] and value vᵢ ∈ [0.02, 0.32] (generated once at startup). An item is selected if expressed[i] > 0.5; otherwise it is left out. The evaluation computes:

W = Σᵢ wᵢ · 𝟙[expressed[i] > 0.5] V = Σᵢ vᵢ · 𝟙[expressed[i] > 0.5]

If total weight W exceeds capacity C = 0.5, fitness is max(0, 1 − 2·(W − C)) — a steep linear penalty. If W ≤ C, fitness is V / V_max where V_max is the sum of all item values. The fitness landscape is inherently discontinuous: a tiny shift in one expressed value can flip a binary decision and cause a large jump in fitness. There are 2¹² = 4096 possible subsets.

Good genes for this problem: Genes that push values decisively toward 0 or 1 are valuable because they make binary decisions cleaner. boundaryPush works against this (it pushes values away from extremes), but bitFlip is useful — it randomly commits some positions to 0 or 1. Conversely, normalize scales values so the largest is always 1.0, which can flip a non-selected item into selected without touching its genome position.

Problematic genes: gradientNudge and sinTransform pull values toward 0.5, which is the decision threshold — this creates ambiguous near-0.5 values that the stochastic evaluation can flip unpredictably. mirrorFold forces half the items to always be the complement of the other half, eliminating half the solution space.

Traveling Salesman — Rank Decoding

There are 10 cities at fixed positions (x, y) ∈ [0,1]². The expressed genome is decoded into a tour by ranking: the city at index i is visited at position rank(expressed[i]) in the tour. Formally, if expressed values are sorted ascending, city i is visited at step k where expressed[i] is the k-th smallest value. The tour length is:

D = Σₖ √((x_{order[k]} − x_{order[k+1]})² + (y_{order[k]} − y_{order[k+1]})²)

where the sum wraps around (the tour returns to the start). Fitness is max(0, 1 − D/5), so a tour of length 5 or more scores 0, and a perfect tour of length 0 would score 1. The critical mathematical property is that only the ordinal ranking of expressed values matters, not their absolute magnitudes — swapping values 0.3 and 0.31 produces the same tour as values 0.001 and 0.999, as long as their relative order is the same.

Good genes for this problem: rankSort is uniquely powerful here because it explicitly converts expressed values into their fractional rank positions, making the genome encoding lossless and maximally informative for the decoder. normalize is similarly good — it stretches the range without changing order. Genes that preserve relative ordering are ideal.

Problematic genes: mirrorFold is highly destructive — it sets expressed[i + half] = 1 − expressed[i], which directly inverts the relative ordering of the second half of the tour. gradientNudge and elitePull push all values toward the centre, reducing spread and making ordinal differences between values very small — neighbouring values become nearly equal and the decoded tour becomes sensitive to floating-point noise.
